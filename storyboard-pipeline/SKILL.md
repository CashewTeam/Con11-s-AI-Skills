# 文案 → 分镜 → Excel 一键管道

## 职责

输入用户文案，在**单次 LLM 调用中**完成分词 + 镜头分配 + 画面编写，输出 Markdown 表格，再调用内置 Python 脚本转换为带颜色标注的 `.xlsx` 文件。

## 技能包结构

```
storyboard-pipeline/
├── SKILL.md              ← 本文件（规则定义）
└── md_table_to_excel.py  ← Python 脚本（Markdown 表格 → Excel）
```

## 第一阶段：分镜生成（单次 LLM 调用）

以下 4 个步骤在同一个 LLM 推理 pass 中完成，保证分词片段与表格行逐字一致。

### 步骤 1：文案分词

对原文案进行切分。

#### 切分原则（优先级从高到低）

| 优先级 | 原则       | 说明                          |
| --- | -------- | --------------------------- |
| 1   | **语义完整** | 每个片段表达一个完整、独立的意思，不在语义单元中间断开 |
| 2   | **长度适中** | 每个片段适合 2～5 秒视频呈现，建议 6～25 个字 |
| 3   | **信息清晰** | 避免单段包含过多信息点，一段一意            |
| 4   | **节奏自然** | 符合口语表达的自然停顿，读起来顺口           |

#### 断点优先级（从强到弱）

1. 句子结束标点（。！？）—— 最强天然断点
2. 从句分隔标点（，；：）—— 次强断点
3. 语义转折词（但是、然而、所以、因此、另外 等）
4. 短语边界（字数达 15+ 且有合理断开处）

#### 铁律

- **绝不改变原文**：仅插入分隔符，不增删改任何字词标点
- 切分后逐段拼接校验，必须与原文逐字一致
- 长度 6～25 字为参考，语义优先：28 字不可拆则不拆，5 字独立语义则独立

#### 特殊场景

| 场景           | 处理               |
| ------------ | ---------------- |
| 文案不足 15 字    | 不切分，整段输出         |
| 列举项（顿号/分号分隔） | 列举引导句 + 各列举项分别成段 |
| 对话/引言        | 说话人切换处必分段        |
| 数据/术语密集      | 适当拆分，数字与修饰对象保持同段 |

### 步骤 2：镜头类型分配

为每一个片段分配镜头类型。必须从以下四大类中选择并标注**具体子类型**，格式为 `大类·子类型`：

#### 实拍

| 子类型    | 说明          |
| ------ | ----------- |
| 推镜头    | 引导注意力聚焦到主体  |
| 拉镜头    | 揭示环境全貌      |
| 摇镜头    | 展示广阔场景或跟随运动 |
| 移镜头    | 轨道平移，营造空间感  |
| 俯拍     | 展示全局布局      |
| 仰拍     | 营造高大、震撼效果   |
| 特写     | 强调细节、质感     |
| 跟拍     | 跟随主体移动      |
| 升格/慢动作 | 强调情绪瞬间      |
| 延时摄影   | 展现时间流逝      |

#### Aroll

| 子类型  | 说明        |
| ---- | --------- |
| 正面口播 | 标准口播      |
| 侧拍口播 | 侧面角度，氛围感  |
| 走动口播 | 边走边说，过渡场景 |
| 情绪口播 | 强化情感，镜头稍近 |

#### 动效

| 子类型    | 说明         |
| ------ | ---------- |
| 2D 动效  | 平面动画、文字动效  |
| 3D 动效  | 三维展示、结构拆解  |
| 后期特效   | 合成特效、粒子、光效 |
| AI 生成  | AI 绘制画面或视频 |
| PPT 动画 | 数据图表、流程图   |
| 关键帧动画  | 属性参数动态演示   |

#### 资料

| 子类型  | 说明        |
| ---- | --------- |
| 历史影像 | 历史事件、人物资料 |
| 新闻素材 | 新闻报道、采访片段 |
| 航拍素材 | 城市风光、自然地貌 |
| 行业图库 | 行业相关配图    |
| 影视片段 | 电影/电视剧引用  |
| 数据图表 | 第三方统计数据   |

#### 分配策略

- 避免连续 3 个以上片段使用同一镜头类型
- 开场和结尾尽量用实拍或 Aroll
- 信息/数据类内容优先动效，场景/情感类优先实拍/Aroll

#### 镜头组合（全局）

| 策略   | 适用场景   | 手法            |
| ---- | ------ | ------------- |
| 快速切换 | 紧张、快节奏 | 片段短、多动效、高频切换  |
| 慢速推进 | 情感渲染   | 升格、长镜头、特写为主   |
| 稳定镜头 | 平静叙述   | 固定机位、Aroll、资料 |
| 运动镜头 | 动态活力   | 跟拍、摇移、推拉      |

叙事逻辑：

- **从大到小**：全景 → 中景 → 特写（建立场景后深入细节）
- **从小到大**：特写 → 中景 → 全景（从细节悬念扩展到环境）
- **对比蒙太奇**：两组反差画面交替（表达矛盾、对比）
- **平行蒙太奇**：多场景并列（展示多样性）

### 步骤 3：分镜内容编写

使用公式：

```
<景别> + <镜头运动> + <画面主体> + <动作/状态> + <氛围/色调>
```

四点要求：

| 要求      | 说明                  |
| ------- | ------------------- |
| **简洁性** | 一句话说清画面，避免冗长        |
| **专业性** | 使用视听语言术语，准确描述镜头和画面  |
| **画面感** | 让读者能立刻浮现画面，描述具体而非抽象 |
| **相关性** | 画面必须与文案内容高度匹配       |

### 步骤 4：生成 Markdown 表格

输出 **6 列** Markdown 表格：

```
| 序号 | 完成 | 原文案 | 镜头类型 | 分镜内容 | 备注 |
|------|------|--------|----------|----------|------|
```

规则：

- `序号`：从 1 开始的数字
- `完成`：**始终留空**，供拍摄时手动标记进度（✓ / 待拍 / 已拍 等）
- `原文案`：步骤 1 切分的片段，与原文逐字一致
- `镜头类型`：步骤 2 分配的结果，如 `实拍·特写`
- `分镜内容`：步骤 3 编写的画面描述
- `备注`：通常留空

在表格后附：

1. 分段说明（解释关键分词断点和镜头分配决策）
2. 镜头类型分布统计

## 第二阶段：Excel 转换

将第一阶段输出的 Markdown 表格写入临时文件，再调用内置脚本生成 Excel。

1. 将 Markdown 表格原样写入 `_storyboard_temp.md`
2. 使用 `RunCommand` 执行：

```powershell
python ./md_table_to_excel.py _storyboard_temp.md -o <用户指定文件名>.xlsx
```

参数：`cwd`: 项目根目录 | `blocking`: `true` | `requires_approval`: `false`

1. 使用 `DeleteFile` 删除 `_storyboard_temp.md`
2. 向用户展示 Excel 绝对路径 + 摘要

### 颜色映射（脚本内置）

| 镜头类型  | 颜色 | RGB      |
| ----- | -- | -------- |
| 实拍    | 红色 | `FFCCCC` |
| Aroll | 绿色 | `CCFFCC` |
| 动效    | 紫色 | `CCCCFF` |
| 资料    | 蓝色 | `CCE5FF` |

## 错误处理

| 异常           | 处理                                |
| ------------ | --------------------------------- |
| 用户未提供文案      | 提示"请提供需要转换为分镜表的文案内容"              |
| 用户未指定文件名     | 默认 `storyboard.xlsx`              |
| Python 脚本失败  | 检查 `_storyboard_temp.md` 格式，修正后重试 |
| 未安装 openpyxl | 执行 `pip install openpyxl`         |

## 注意事项

- 镜头类型必须标注子类型，不可只写大类（如只写"实拍"）
- 分镜内容不可为空
- 文案用途（广告/教学/解说）影响镜头风格和目标受众感
- 分词铁律：绝不改变原文任何字词标点
